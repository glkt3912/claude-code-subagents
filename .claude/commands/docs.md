# Documentation Generator

Generate well-structured documentation for projects, APIs, or features.

## Usage

```
/docs [doc_type] [target]
```

## Examples

- `/docs readme` - Generate README.md
- `/docs api user-service` - Generate API documentation
- `/docs guide setup` - Generate setup guide

## Documentation Structure

1. **Overview**: Description, use cases, target audience
2. **Getting Started**: Installation, configuration, quick start
3. **Usage**: Detailed examples and common patterns
4. **Configuration**: Available options and settings
5. **Troubleshooting**: Common issues and solutions
6. **Contributing**: Development and submission guidelines

## Standards

- Proper heading hierarchy (H1 → H2 → H3)
- Code language specifications
- Practical examples for all concepts
- Consistent formatting and terminology
- Markdown compliance
